# Академия тенниса

Игровой тренажёр по тактике тенниса с маскотом Эйсом 🎾. 116 вопросов в 17 блоках, очки-мячи, звания, «Тренировка дня» и вторая подача. Поддерживает многопользовательский режим через Firebase Firestore.

## Как открыть (локально)

Запустите локальный сервер (Firebase-SDK требует HTTP, не `file://`):

```
/usr/local/bin/python3.13 -m http.server 8000
```

Затем откройте `http://localhost:8000` в браузере.

Без настоящего Firebase-конфига игра работает полностью автономно — без регистрации и стены.

## Механика

### Гейм
- Каждый блок или «Тренировка дня» — гейм из 7 вопросов (меньше, если в блоке меньше).
- Отбор умный: сначала незнакомые (корзина A) → ранее ошибочные (B) → уже освоенные (C).
- Лёгкий старт: первыми идут до 2 вопросов базового (Б) уровня.
- Сортировка внутри гейма: Б → С → П.

### Вторая подача
- Неверный ответ с первой попытки → вопрос добавляется в конец очереди (вторая подача).
- Верный со второй — начисляется половина очков; точка становится зелёной.
- Неверный со второй — очков нет, точка остаётся жёлтой, вопрос попадает в «Разобрать ошибки».

### Очки 🎾
| Сложность | 1-я подача | 2-я подача |
|-----------|-----------|-----------|
| Б (базовый) | 10 | 5 |
| С (средний) | 20 | 10 |
| П (продвинутый) | 30 | 15 |

- **Серия**: при 3 верных подряд включается множитель ×2 (с 4-го ответа).
- Серия сбрасывается при ошибке; очки при этом не сгорают.

### Звания
| Порог | Звание | Эмодзи |
|-------|--------|--------|
| 0     | Новичок | 🎒 |
| 300   | Юниор | 🥎 |
| 800   | Игрок основы | 🎾 |
| 1600  | Капитан команды | 🏅 |
| 2800  | Мастер корта | 🏆 |
| 4500  | Чемпион Академии | 👑 |

### Звёзды блока
- ★ — отвечено верно хотя бы раз ≥ 50% вопросов блока.
- ★★ — ≥ 80%.
- ★★★ — 100%.

## localStorage: схема профиля

Ключ: `tennisAcademy.v1`

```json
{
  "v": 1,
  "points": 420,
  "onboarded": true,
  "history": {
    "1": { "c": 2, "w": 0, "last": 1, "t": 1718000000000 },
    "5": { "c": 0, "w": 1, "last": 0, "t": 1718001000000 }
  }
}
```

- `points` — общий счёт (целое ≥ 0).
- `onboarded` — показан ли онбординг (boolean).
- `history` — история по каждому вопросу (ключ = строка `q.id`):
  - `c` — количество верных ответов.
  - `w` — количество неверных ответов.
  - `last` — результат последней попытки: 1 = верно, 0 = неверно.
  - `t` — `Date.now()` последнего ответа.

Аккаунт Firebase: ключ `tennisAcademy.account` — `{ uid, name }`.  
Офлайн-очередь: ключ `tennisAcademy.syncQueue` — массив событий для досылки.

## Как сбросить прогресс

Откройте DevTools браузера → вкладка **Application** → **Local Storage** → удалите ключ `tennisAcademy.v1`. После перезагрузки страницы начнётся онбординг.

Или в консоли:
```js
localStorage.removeItem('tennisAcademy.v1');
location.reload();
```

## Как перегенерировать данные

Если вы изменили `tennis_question_bank.md`, пересоздайте `questions.js`:

```
/usr/local/bin/python3.13 convert.py
```

Скрипт выведет `OK: 116 вопросов, 17 блоков` и обновит файл `questions.js`.

## Многопользовательский режим (Firebase)

### Схема данных Firestore

**Коллекция `players/{uid}`:**
```
uid           — uid Firebase Auth (Email/Password, стабилен на всех устройствах)
name          — имя игрока (1–20 символов)
points        — всего мячей
rankName      — текущее звание (строка с эмодзи)
blockStars    — { "0": 3, "1": 1, ... } — звёзды 17 блоков
bestStreakEver — рекордная серия
daysCount     — количество дней тренировок
weeklyPoints  — очки за текущую неделю (ISO 8601)
weekKey       — ключ недели, напр. "2025-W23"
createdAt     — serverTimestamp
updatedAt     — serverTimestamp
```

**Коллекция `feed/{autoId}`:**
```
uid    — uid автора
name   — имя автора
type   — "block_stars" | "rank_up" | "daily_done"
text   — текст события
emoji  — один эмодзи
ts     — serverTimestamp
```

История вопросов (`history`), значки (`badges`) и список дней (`days`) хранятся **только в localStorage** и в облако не синхронизируются.

### Пошаговая инструкция для владельца

#### 1. Создайте Firebase-проект

1. Перейдите на https://console.firebase.google.com/ и создайте новый проект.
2. В разделе **Authentication** включите провайдер **Email/Password** (Sign-in method).
3. В разделе **Firestore Database** нажмите «Создать базу данных» (режим Production).

#### 2. Вставьте конфиг в firebase-config.js

1. В консоли Firebase: «Настройки проекта» → «Ваши приложения» → «+ Добавить приложение» → Web.
2. Скопируйте объект `firebaseConfig`.
3. Откройте `firebase-config.js` и замените `var FIREBASE_CONFIG = null;` на:

```js
var FIREBASE_CONFIG = {
  apiKey: 'AIzaSy...',
  authDomain: 'ваш-проект.firebaseapp.com',
  projectId: 'ваш-проект',
  storageBucket: 'ваш-проект.appspot.com',
  messagingSenderId: '123456789',
  appId: '1:123456789:web:abcdef'
};
```

#### 3. Опубликуйте правила Firestore

В терминале (нужен Firebase CLI):

```bash
npm install -g firebase-tools
firebase login
firebase init firestore   # выберите существующий проект
# скопируйте содержимое firestore.rules в созданный файл
firebase deploy --only firestore:rules
```

Или вставьте содержимое `firestore.rules` напрямую в редакторе правил в консоли Firebase.

#### 4. Публикация на GitHub Pages

1. Создайте репозиторий на GitHub и загрузите файлы проекта.
2. В настройках репозитория: **Settings → Pages → Branch: main → / (root)**.
3. В консоли Firebase: **Authentication → Settings → Authorized domains** — добавьте домен вида `ваш-логин.github.io`.
4. Игра доступна по адресу `https://ваш-логин.github.io/имя-репозитория/`.

### Аутентификация

- **Синтетический Email/Password**: вход реализован через Firebase Email/Password. Email строится как `<localpart>@academy.local`, где localpart — нормализованное имя (trim → lowercase → ё→е → транслитерация → не [a-z0-9] заменяются точкой). Пароль = ПИН + суффикс `Ac` (Firebase требует ≥ 6 символов).
- **Стабильный uid**: один и тот же игрок получает один uid на всех устройствах. Правила Firestore (`auth.uid == uid`) работают без изменений.
- **Глобально уникальные имена**: нормализация гарантирует, что «Артём» и «артем» — один аккаунт.
- **Вход с другого устройства**: вкладка «У меня есть аккаунт» на экране входа — достаточно ввести имя и ПИН.
- **ПИН не восстанавливается** (by design): если забыл — заведи новый аккаунт.
- **pinHash в Firestore больше не хранится**: переход на Firebase Auth, хранение хеша паролей на стороне Firebase.
- **Защита от перебора**: Firebase возвращает `auth/too-many-requests` при частых неудачных попытках; дополнительно клиент блокирует кнопку входа на 30 секунд после 5 неудач.
- **Предусловие**: в Firebase Console должен быть включён провайдер **Email/Password** (Authentication → Sign-in method).

### Честные ограничения безопасности

- **ПИН — «детский замок»**: 4-значный ПИН даёт 10 000 комбинаций. Это защита от случайного входа под чужим именем, но не от целенаправленного перебора.
- **Накрутка очков не предотвращается**: игра работает на клиенте, проверки на сервере нет. Рейтинг показывает прогресс участников честной игры, а не защищённый чемпионат.

## Структура проекта

- `tennis_question_bank.md` — исходный банк вопросов (не редактировать)
- `convert.py` — конвертер из markdown в JS
- `questions.js` — сгенерированные данные (не редактировать)
- `index.html` — разметка (7 экранов)
- `style.css` — игровые стили
- `app.js` — логика Академии
- `firebase-config.js` — конфиг Firebase (проект tennis-academy-2026; `null` = локальный режим)
- `cloud.js` — сетевой слой Firebase (инициализация, синк, стена, рейтинг)
- `firestore.rules` — правила безопасности Firestore
