# Академия тенниса 🎾

Детский веб-тренажёр по тактике тенниса с маскотом Эйсом 🎾. Vanilla HTML/CSS/JS без сборки, русский интерфейс, mobile-first. **116 вопросов в 17 блоках**: очки-мячи, звания, серии, «Тренировка дня», вторая подача, звёзды блоков и значки. Есть многопользовательский режим — общая «Стена академии» и недельный рейтинг.

**Играть:** https://letshavefun8.github.io/tennis-academy/

## Как открыть локально

Игра — это статические файлы. Запустите любой локальный сервер из корня репозитория:

```
python3 -m http.server 8000
```

Затем откройте `http://localhost:8000`.

Если облачный бэкенд недоступен (или адрес в `cloud.js` равен `null`), игра работает полностью автономно: проходить геймы, копить очки и звания можно без регистрации, только без Стены и рейтинга.

## Механика игры

### Гейм
- Каждый блок или «Тренировка дня» — гейм из 7 вопросов (меньше, если в блоке меньше).
- Отбор умный: сначала незнакомые (корзина A) → ранее ошибочные (B) → уже освоенные (C).
- Лёгкий старт: первыми идут до 2 вопросов базового (Б) уровня.
- Сортировка внутри гейма: Б → С → П.

### Вторая подача
- Неверный ответ с первой попытки → вопрос добавляется в конец очереди (вторая подача).
- Верный со второй — начисляется половина очков; точка становится зелёной.
- Неверный со второй — очков нет, точка остаётся жёлтой, вопрос попадает в «Разобрать ошибки».

### Очки 🎾
| Сложность | 1-я подача | 2-я подача |
|-----------|-----------|-----------|
| Б (базовый) | 10 | 5 |
| С (средний) | 20 | 10 |
| П (продвинутый) | 30 | 15 |

- **Серия**: при 3 верных подряд включается множитель ×2 (с 4-го ответа).
- Серия сбрасывается при ошибке; накопленные очки при этом не сгорают.

### Звания
| Порог | Звание | Эмодзи |
|-------|--------|--------|
| 0     | Новичок | 🎒 |
| 300   | Юниор | 🥎 |
| 800   | Игрок основы | 🎾 |
| 1600  | Капитан команды | 🏅 |
| 2800  | Мастер корта | 🏆 |
| 4500  | Чемпион Академии | 👑 |

### Звёзды блока
- ★ — отвечено верно хотя бы раз на ≥ 50% вопросов блока.
- ★★ — ≥ 80%.
- ★★★ — 100%.

## Архитектура

Два независимых слоя.

### Фронтенд — GitHub Pages
Статика (`index.html`, `style.css`, `app.js`, `cloud.js`, `questions.js`) хостится на GitHub Pages по адресу https://letshavefun8.github.io/tennis-academy/. Сборки нет — браузер грузит файлы как есть. Прогресс хранится локально в `localStorage`, агрегаты профиля и события Стены отправляются в облако.

### Бэкенд — Yandex Cloud
Изначально многопользовательский режим работал на Firebase Firestore. **Firebase блокируется в РФ без VPN**, поэтому бэкенд переехал на **Yandex Cloud**, который доступен в РФ без VPN. Текущий стек:

- **Cloud Function** (Python, `backend/handler.py`) — вся серверная логика в одной функции.
- **YDB через Document API** (DynamoDB-совместимый, `boto3`) — хранилище.
- **API Gateway** — даёт публичный HTTPS-адрес и проксирует все запросы в функцию (см. `backend/api-gateway.yaml`, маршрут `/{path+}` на одну функцию, она сама роутит по пути).

Боевой адрес бэкенда задан в `cloud.js` в переменной `CLOUD_BACKEND_URL`. `cloud.js` — это сетевой слой: обычный `fetch` к API Gateway, токен-авторизация, лента через polling (раз в 15 с, `FEED_POLL_MS`), офлайн-очередь feed-событий с досылкой при возвращении сети.

### Авторизация
Своя, без внешнего провайдера:

1. Клиент шлёт **имя + 4-значный ПИН** на `/register` или `/login`.
2. Сервер хеширует ПИН (`SHA-256` с серверным секретом `APP_SECRET`) и сверяет с хранимым `pin_hash`.
3. При успехе выдаёт **HMAC-токен сессии** (`make_token`, подпись `APP_SECRET`, TTL — год).
4. Токен хранится в `localStorage` и прикладывается к запросам на `/sync` и `/feed` (POST).

Имена нормализуются (`trim → lowercase → ё→е → схлопывание пробелов`) и глобально уникальны — «Артём» и «артем» считаются одним игроком. Это позволяет входить с любого устройства по имени и ПИН.

### Таблицы YDB
- **`players`** — профили: `uid`, `name`, `name_key`, `pin_hash`, `points`, `rank_name`, `block_stars` (JSON), `best_streak`, `days_count`, `weekly_points`, `week_key` (ISO-неделя, напр. `2026-W24`), `created_at`, `updated_at`.
- **`names`** — уникальность имён и вход по имени: `name_key → uid` (резервируется атомарно через `attribute_not_exists`).
- **`feed`** — лента/Стена: партиция `bucket="all"`, сортировочный ключ `ts_id` (время + случайный суффикс), поля `uid`, `name`, `type`, `text`, `emoji`, `ts`.

Что **не** синхронизируется в облако (только `localStorage`): история ответов по каждому вопросу, значки, точный список дней. На сервер уходят агрегаты профиля (`buildAggregates` в `cloud.js`).

## API эндпоинты

Все запросы идут на адрес API Gateway; функция маршрутизирует по последнему сегменту пути и отдаёт CORS-заголовки (включая ответ на `OPTIONS`-предзапрос).

| Метод | Путь | Назначение |
|-------|------|-----------|
| POST | `/register` | Регистрация: `{name, pin, ...агрегаты}` → `{ok, uid, name, token}`. Резервирует имя, создаёт профиль, мигрирует локальный прогресс. |
| POST | `/login` | Вход: `{name, pin}` → `{ok, uid, name, token, player}`. Проверяет хеш ПИН. |
| POST | `/sync` | Синхронизация агрегатов: `{token, points, rankName, blockStars, bestStreakEver, daysCount, sessionPoints}`. Пересчитывает недельные очки. |
| POST | `/feed` | Публикация события в Стену: `{token, type, text, emoji, name}`. Допустимые `type`: `block_stars`, `rank_up`, `daily_done`, `lesson_done`. |
| GET  | `/feed` | Последние 30 событий Стены (по убыванию времени). |
| GET  | `/leaderboard` | Топ-20 игроков по очкам за текущую неделю. |
| GET  | `/player?uid=` | Публичный профиль игрока по `uid` (без `pin_hash`). |

## Структура проекта

```
index.html            — разметка (несколько экранов), нижний таб-бар, ссылки на ассеты с ?v=N

cloud.js              — сетевой слой: fetch к API Gateway, токен-авторизация, polling-лента, офлайн-очередь

convert.py            — конвертер банка вопросов в questions.js
design-preview.html   — эталон дизайна (референс)
backend/
  handler.py          — Cloud Function (вся серверная логика)
  requirements.txt    — зависимости функции (boto3)

```

## Разработка и деплой

### Регенерация вопросов
`questions.js` генерируется из `tennis_question_bank.md`. После правки банка пересоберите данные:

```
python3 convert.py
```

Скрипт выведет `OK: 116 вопросов, 17 блоков` и перезапишет `questions.js`. Сам `questions.js` руками не редактируется.

### Версионирование ассетов (защита от кеша)
В `index.html` ссылки на статику имеют вид `app.js?v=N`. При каждом деплое фронта номер `N` бампается у всех ассетов — иначе браузеры отдадут закешированную старую версию. Параллельно обновляется константа `APP_VERSION` в `app.js` (она видна внизу экрана и помогает понять, какая сборка открыта). На момент написания — сборка 11.

### Деплой фронтенда
```
git push        # GitHub Pages сам пересобирает страницу из ветки main
```

### Деплой бэкенда
Через Yandex Cloud CLI из каталога `backend/`:

```
yc serverless function version create ...   # новая версия Cloud Function
```

API Gateway настраивается из `backend/api-gateway.yaml` (привязка к function_id и сервисному аккаунту). Секреты (`DOCAPI_ENDPOINT`, `APP_SECRET`, ключи доступа к облаку) задаются переменными окружения функции, а не лежат в коде.

### Секреты
Ключи Yandex Cloud и `APP_SECRET` **не хранятся в репозитории** — реальные значения лежат локально в `~/`. В `.gitignore` исключены `*-key.json` и `*.env`.

## Сброс локального прогресса

Локальный профиль лежит в `localStorage` под ключом `tennisAcademy.v1`. Сбросить:

```js
localStorage.removeItem('tennisAcademy.v1');
location.reload();
```

Или через DevTools → **Application → Local Storage** → удалить ключ. После перезагрузки начнётся онбординг. Аккаунт хранится отдельно (`tennisAcademy.account`), офлайн-очередь Стены — в `tennisAcademy.syncQueue`.

## Честные ограничения

- **ПИН — это «детский замок».** 4 цифры дают 10 000 комбинаций: защита от случайного входа под чужим именем, но не от целенаправленного перебора. Забытый ПИН не восстанавливается (by design) — заводится новый аккаунт.
- **Накрутка очков на клиенте не предотвращается.** Игровая логика и подсчёт очков работают в браузере, сервер принимает присланные агрегаты на доверии. Рейтинг отражает прогресс честно играющих участников, а не защищённый чемпионат.
- **Данные хранятся в облаке на территории РФ** (Yandex Cloud). Это сознательный выбор ради доступности без VPN. Профили детей минимальны: имя (часто игровое) и игровые агрегаты, никаких email и персональных данных.
