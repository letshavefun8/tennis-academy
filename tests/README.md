## Команды запуска

### P0 — данные (Python, без браузера)
```
python3 tests/run.py
```
Ожидаемый вывод: все строки `PASS`, последняя строка `N passed, 0 failed`, код выхода 0.

### P1 — логика (браузер, через http-сервер)
```bash
# Запустить сервер из корня репо:
python3 -m http.server 8000

# Открыть в браузере:
# http://localhost:8000/tests/browser.html
```
Ожидаемый результат: в заголовке вкладки `✓ N/N — tennis tests PASS`, все строки с `✓`.

> Открывать строго через `http://`, а не через `file://` — иначе браузер заблокирует загрузку скриптов из соображений безопасности.

## P2 — отложенный кейс: очки и серия ×2 (НЕ реализован как тест)

Логика вшита в `handleAnswer` (строка ~893 app.js), который зависит от DOM и сессии.
Вынести в юнит-тест без рефакторинга нельзя — оставлено для будущего.

### Числовой пример для ручной проверки:
- Вопрос сложности **Б**, `basePoints = 10`.
- `streak` до ответа: 0, 1, 2 → множитель `1` (streak < 3 до инкремента).
- 4-й верный ответ подряд: на момент вызова `handleAnswer` streak = 3 → `multiplier = 2` → earned = `10 × 2 = 20`. Затем `streak++` → streak = 4.
- Итого 4 верных Б подряд: `10 + 10 + 10 + 20 = 50 очков`.
- Неверный ответ: `earned = 0`, `streak = 0`, ранее заработанные очки **не сгорают**.
- Retry-верный (повторная попытка): `earned = floor(base / 2) = 5` (для Б).

Формула: `multiplier = (session.streak >= 3) ? 2 : 1`, затем `session.streak++`.
Проверять: открыть игру, сыграть 4+ верных ответа подряд, убедиться в ×2 с 4-го.
